Git基本操作

基本操作

检查当前文件状态

使用git status命令可以列出工作区、暂存区和本地仓库之间状态不同的文件，如果使用了-ignored选项，还可以列出工作区中已忽略的文件，主要包含以几种：

  • Changes to be committed：指暂存区与本地仓库之间存在差异的文件列表
    • commit命令：暂存区=>本地仓库
    • restore –staged命令：本地仓库=>暂存区
  • Changes not staged for commit：指工作区与暂存区之间存在差异的文件列表
    • add 命令：工作区=>暂存区
    • restore命令：暂存区=>工作区
  • Untracked files：未追踪的文件，通过add命令添加到暂存区，如果文件在.gitignore中设置为忽略，则add命令不会添加
  • Ignored files：使用–ignored选项后列出以忽略的文件，可以通过-f选项强制添加到暂存区

使用-s选项可以列出文件状态简写模式，包括??，A，M，D，等，??表示Untracked files，A表示新增文件，M表示文件修改，D表示文件被删除。输出的第一列是暂存区与本地仓库之间相比较，第二列是工作区与暂存区之间相比较。

添加：工作区=>暂存区

git add .命令可以更新工作区中的所有文件，包括已删除的文件和新增的文件，也可以通过git add file 或git add *.py添加和匹配特定的文件。

  • -n，–dry-run选项：用于测试，输出该命令会add的文件列表，但是实际上不会添加文件

  • -u，–update选项：更新文件，不包括Untracked files

  • -A，–all选项：添加所有文件，与git add .相似

  • -f选项：强制添加已忽略的文件

删除和移动文件

普通的删除命令和移动命令只能改变工作区的内容，git rm file和git mv file同时在工作区和暂存区中进行删除和移动，相当于执行删除和移动指令后再自动执行add命令

git rm –cached可以仅删除暂存区文件而保留工作区。

提交：暂存区=>本地仓库

git commit命令将暂存区内容提交到本地仓库，每一次提交都会产生一个记录，不像工作区和暂存区都是临时的内容。

  • -m选项：为commit添加注释
  • -a选择：相当于提交前自动执行git add -u，不能添加新增文件
  • -amend选项：自动打开编辑器，在交互界面修改，并重新提交

使用amend选项不仅修改了提交信息，还自动修改了提交时间，第一行为创建commit的时间，第二行为最后一次提交的时间。

撤销：本地仓库=>暂存区=>工作区

git restore命令可以将暂存区的内容替换工作区的内容，使用–staged选项可以将当前本地仓库的内容（HEAD）替换暂存区的内容。

git reset commit-hash命令可以将HEAD指针移动到指定的commit上，通常用于撤销commit，同时还可以替换暂存区和工作区，具有以下模式

  • –hard：同时重置暂存区和工作区
  • –soft：既不重置暂存区也不重置工作区
  • –mixed(默认)：只重置暂存区，保留工作区
  • –merge：暂未使用
  • –keep：重置暂存区和工作区，但是如果文件在本地修改了，则终止reset

分支管理

创建和删除分支

  • git branch dev：在当前HEAD处创建分支dev
  • git branch dev commit-hash：在指定commit处创建分支dev
  • git branch -m new：分支重命名（当前分支=>new）
  • git branch -m old new：分支重命名（old=>new）
  • git branch -d dev：删除分支dev
  • git branch -D dev：强制删除未合并的分支dev
  • git branch -c dev：复制分支（复制reflog）

远程仓库

添加和查看远程仓库

  • git remote add [name] [url]：添加远程仓库
  • git remote show [name]：查看远程仓库
